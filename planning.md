### Tool 1: search_listings

**What it does:**
<!-- Describe what this tool does in 1–2 sentences -->
Searches the clothing marketplace dataset and returns listings that match the user's requested item description, size, and budget.

**Input parameters:**
<!-- List each parameter, its type, and what it represents -->
- `description` (str): Keywords describing the item the user wants.
- `size` (str): Desired clothing size.
- `max_price` (float):  Maximum budget.

**What it returns:**
<!-- Describe the return value — what fields does a result contain? -->
A list of matching listings sorted by relevance. 

Each listing contains:
id (int)
title (str)
description (str)
category (str)
style_tags (list[str])
size (str)
condition (str)
price (float)
colors (list[str])
brand (str)
platform (str)

**What happens if it fails or returns nothing:**
<!-- What should the agent do if no listings match? -->
The planning loop stops immediately and returns a message explaining that no matching listings were found. The user is given suggestions such as increasing their budget, changing keywords, or trying a nearby size.

---

### Tool 2: suggest_outfit

**What it does:**
<!-- Describe what this tool does in 1–2 sentences -->
Generates a styling recommendation by combining the selected listing with items from the user's wardrobe.

**Input parameters:**
<!-- List each parameter, its type, and what it represents -->
- `new_item` (dict): Listing selected from search results.
- `wardrobe` (dict): User wardrobe loaded from the wardrobe schema.

**What it returns:**
<!-- Describe the return value -->
A natural-language outfit recommendation describing how to style the selected item using wardrobe pieces.

**What happens if it fails or returns nothing:**
<!-- What should the agent do if the wardrobe is empty or no outfit can be suggested? -->
If the wardrobe is empty, the tool returns a generic styling recommendation based on the item's style tags instead of a personalized outfit.

---

### Tool 3: create_fit_card

**What it does:**
<!-- Describe what this tool does in 1–2 sentences -->
Creates a short social-media-style caption based on the selected item and outfit recommendation.

**Input parameters:**
<!-- List each parameter, its type, and what it represents -->
- `outfit` (str):Outfit recommendation generated by suggest_outfit().
new_item (dict): Selected clothing item.

**What it returns:**
<!-- Describe the return value -->
A short caption suitable for Instagram, TikTok, or Depop outfit posts.

**What happens if it fails or returns nothing:**
<!-- What should the agent do if the outfit data is incomplete? -->
The planning loop skips fit card generation and returns the listing and outfit recommendation only.

## Planning Loop

**How does your agent decide which tool to call next?**
<!-- Describe the logic your planning loop uses. What does it look at? What conditions change its behavior? How does it know when it's done? -->

1. Receive user query.
2. Extract search terms, size, and budget.
3. Call search_listings(description, size, max_price).
4. If results are empty:
      Set error message in session state.
      Return response immediately.
5. Otherwise:
     Set selected_item = results[0].
6. Load wardrobe.
7. Call suggest_outfit(selected_item, wardrobe).
8. Store result as outfit_suggestion.
9. Call create_fit_card(outfit_suggestion, selected_item).
10. Store result as fit_card.
11. Return:
     Selected listing
     Outfit recommendation
     Fit card

The workflow ends after the fit card is generated or after an error condition causes an early return.

## Error Handling

For each tool, describe the specific failure mode you're handling and what the agent does in response.

| Tool | Failure mode | Agent response |
|------|-------------|----------------|
| search_listings | No results match the query | Return: "I couldn't find any matching items. Try increasing your budget, changing keywords, or searching nearby sizes."|
| suggest_outfit | Wardrobe is empty | Generate a generic outfit recommendation using item style tags instead of wardrobe data.|
| create_fit_card | Outfit input is missing or incomplete | Skip fit card generation and return listing plus outfit recommendation only.|
